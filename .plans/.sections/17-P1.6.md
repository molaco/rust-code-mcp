### P1.6 — Guideline gates + boundary allowlist

**Goal:** A `GuidelineGate` harness that, after a CRUD patch lands on the working snapshot, runs the existing audits scoped to the dirty set and classifies every violation as a hard `Refusal` (roll back) or a soft penalty (apply + reward signal), with a human-curated, agent-read-only `BoundaryAllowlist`.

**Key logic / algorithm:**
- The harness is a *post-apply* check: `CrudEngine` (P1.5a) calls `GuidelineGate::check` after `WorkspaceHost::apply` has patched the working snapshot, passing the op's `AffectedSet` (P0.2/D2). From the affected set we take the editing crate's `NodeId` as `crate_id_filter` and the dirty `FileId`s to build `only_files`, so the audit cost is proportional to the edit, not the workspace (resolves Issue: "audit latency at write time").
- Pattern violations reuse the eight built-in matchers verbatim through `fn_body_audit::fn_body_audit_scoped` (`match_unwrap`, `match_expect`, `match_panic_macros`, `match_unwrap_unchecked`, `match_unbounded_loop`, `match_await_in_guard_scope`, `match_transmute`, `match_self_recursion`); each `FnBodyFinding.pattern` string maps through `classify()` to `Hard`/`Soft`. `unsafe` violations come from `graph::unsafe_audit::unsafe_audit_impl` (one `UnsafeFinding` per `unsafe {}`), gated only when *not* `has_safety_comment` and *not* allowlisted.
- Numeric body guidelines (cyclomatic / param count / body LOC / nesting) are NOT in the current audits — `compute_body_metrics` adds them. It reads `OpenedSnapshot::function_signature(target).params.len()` for the param count (no AST needed) and walks the `ast::Fn` body the same way `fn_body_audit` already parses files (`sema.parse_guess_edition` → `descendants` → `ast::Fn::cast` → `body()`). The decision-point count is the AST equivalent of the regex heuristic in `analyze_complexity` (analysis.rs:477-487) but node-based, not line-based, so it is robust to formatting.
- The cycle (P3) and visibility-widening (P13) refusal *reasons* live in `RefusalReason` here, but are *produced* by the CRUD ops: P1.5b's `cycle_gate::assert_no_new_cycle` (via `crate_edges()` SCC) constructs `RefusalReason::Cycle`, and a future visibility op consults `dead_pub_in_crate`/`BindingVisibility`. The gate centralizes only the type + JSON shape + severity so every refusal is uniform training data.
- The `BoundaryAllowlist` is loaded once from a workspace TOML (`[[boundary]] target/reason`, per design), held immutable in `GuidelineGate`; `excuses()` is consulted before promoting an `Unsafe`/`Transmute` finding to a `Refusal` (the only sanctioned bypass). No agent-facing mutation method is compiled, satisfying governing principle #2 ("the agent never bypasses guidelines").
- `GateOutcome::Refused` signals `CrudEngine` to roll back the working-snapshot patch via the D4 undo marker (no source left mutated); `Pass{penalties}` flows the soft `Penalty` list into P1.7's `RewardVector.principle_violations`. Compile/test gates (`CompileBreak`/`TestFailure`) are *reserved* variants here but actually evaluated in P1.7 (cargo/RA gate) — P1.6 owns only the static, dirty-scoped audits.

**Depends on:** [P0.2 (AffectedSet, WorkspaceHost, OpenedSnapshot dirty file-ids), P1.1 (Location→NodeId, LoadedWorkspace access)]. Consumed by: [P1.5a–e (CrudEngine gate hook), P1.7 (Penalty → RewardVector)].

**Risks / unknowns:**
- Threshold calibration is unspecified by the design — the `GateConfig` defaults (max_cyclomatic/params/LOC/nesting) are guesses (verify) and likely need tuning against the rmc repo's own functions so baseline code is not all-refused.
- `compute_body_metrics` nesting/cyclomatic on RA AST must agree with whatever Phase-2 reward shaping expects; the existing `analyze_complexity` is a line-regex on a *single file path* (not graph-scoped) and is not reusable directly — this is net-new AST work, not a wrapper.
- `fn_body_audit_scoped` must keep the same enclosing-fn resolution (`enclosing_fn_for_body_offset`) so findings still attach a `NodeId`; restricting `file_ids` must not change per-finding output vs the full audit (differential test against `fn_body_audit` on the same single file).
- Allowlist matching semantics (exact qualified-name vs glob vs module-prefix) are undecided; a too-broad glob silently excuses real violations — start with exact `==` + explicit `::*` suffix, no regex (verify).
- A body edit that *moves* a guideline violation out of the dirty fn but into a caller (e.g. unwrap relocated) won't be seen by a dirty-scoped gate; per "right-sized rigor" the per-episode `declare_done` cargo/audit pass (P1.7) is the backstop, but mid-episode the agent could accrue an unseen violation. (accept; document.)
- Self-recursion (`match_self_recursion`) needs the enclosing fn's qualified name; on a freshly-edited body the warm-host re-extract must have refreshed it before the gate runs, else the recursion matcher silently no-ops.
