### P1.4 — Counterfactual simulator
**Goal:** `Query::simulate(op)` returns predicted audit/metric deltas + cascade + would-refuse **without** mutating source or LMDB, by running the *exact same* effect computation `Crud::apply` uses so the prediction cannot diverge from the real apply.

**Types** (new/mod/del):
- `Effects` @ rmc-env::crud::effects — struct (new) — the pure result of planning an op: `{ source_edits: Vec<SourceEdit>, edit_class: EditClass, affected: AffectedSet, patch: GraphPatch, refusal: Option<Refusal> }`. Produced by `compute_effects`; consumed by both `persist` and `predict`.
- `SimResult` @ rmc-env::sim — struct (new) — `{ would_refuse: Option<Refusal>, audit_delta: AuditDelta, metric_delta: MetricDelta, cascade: Cascade, edit_class: EditClass }`. The counterfactual answer surfaced in the `ContextView`.
- `Simulator` @ rmc-env::sim — struct (new) — holds `&WorkspaceHost` + `&CrudEngine`; entrypoint `simulate`.
- `AuditDelta` @ rmc-env::sim::predict — struct (new) — per-audit before/after counts: `{ unwrap_panic: i32, unsafe_blocks: i32, recursion_cycles: i32, mut_statics: i32, dead_pub: i32, complexity_change: i32 }` (signed; mirrors the reward vector's `audit_delta`).
- `MetricDelta` @ rmc-env::sim::predict — struct (new) — `{ instability_change: f64, abstractness_change: f64, modularity: f64, conductance: f64 }` (the P1.7 graph-metric subset that is cheap to recompute on the affected set).
- `Cascade` @ rmc-env::sim::predict — struct (new) — `{ affected_crates: Vec<String>, callers_touched: Vec<Location>, reverse_dep_items: u32 }` — what the op would ripple into (the D2 affected-set rendered for the agent).
- `SourceEdit` @ rmc-env::crud::effects — struct (new) — `{ file: PathBuf, span: (u32,u32), replacement: String }` — one buffer edit the op would make (verify: may already exist in P1.5 CrudEngine; reuse if so).

**Functions / methods** (new/mod/del):
- `fn compute_effects(&self, host: &WorkspaceHost, action: &Crud) -> Result<Effects, CrudError>` @ rmc-env::crud::engine — new — plans the op: builds `SourceEdit`s, classifies `EditClass`, computes `AffectedSet` (D2) and `GraphPatch` (D3), and runs the **hard-refusal gates** (P1.6) against the *would-be* graph state; returns early with `Effects { refusal: Some(..) }` if refused. **Performs no source writes, no `set_file_text`, no LMDB `put`.**
- `fn persist(&self, host: &mut WorkspaceHost, effects: Effects) -> Result<AppliedOp, CrudError>` @ rmc-env::crud::engine — new — the only side-effecting half: writes `source_edits` to disk, `host.apply_edits()` (`set_file_text` + scoped `emit_crate_scoped`), `working.apply_patch(effects.patch)`, records the `UndoLog` entry.
- `fn apply(&self, host: &mut WorkspaceHost, action: &Crud) -> Result<AppliedOp, CrudError>` @ rmc-env::crud::engine — mod — now exactly `self.persist(host, self.compute_effects(host, action)?)`. This is the anti-lie guarantee: `simulate` and `apply` share `compute_effects` verbatim.
- `fn simulate(&self, action: &Action) -> Result<SimResult, SimError>` @ rmc-env::sim::Simulator — new — for a `Crud` (or simulatable `Query`) op: `let eff = self.crud.compute_effects(self.host, op)?;` then `predict::from_effects(self.host, &eff)`; returns `SimResult` with `would_refuse = eff.refusal`. Never calls `persist`.
- `fn from_effects(host: &WorkspaceHost, eff: &Effects) -> Result<(AuditDelta, MetricDelta, Cascade), SimError>` @ rmc-env::sim::predict — new — opens an **overlay** read view (`WorkingSnapshot::overlay_txn`) with `eff.patch` applied-but-uncommitted, runs the affected-set-scoped audits/metrics on it, diffs vs the pre-op baseline.
- `fn overlay_txn(&self, patch: &GraphPatch) -> Result<OverlaySnapshot<'_>, HostError>` @ rmc-host::snapshot::WorkingSnapshot — new — begins an LMDB `RwTxn`, applies `patch` into it, hands back an `OpenedSnapshot`-compatible read view bound to that txn, and **drops the txn without commit** on `Drop` (LMDB rolls back). Lets the existing `query/*` methods run against counterfactual state with zero persistence.
- `fn predict_audit_delta(snap_before, snap_after, scope: &AffectedSet) -> AuditDelta` @ rmc-env::sim::predict — new — calls scoped `fn_body_audit` (unwrap/panic count), `unsafe_audit`, `recursion_check`, `mut_static_audit`, `dead_pub_in_crate` before/after and subtracts.
- `fn predict_metric_delta(snap_before, snap_after, scope) -> MetricDelta` @ rmc-env::sim::predict — new — calls `crate_dependency_metric` (instability/abstractness) before/after on affected crates; computes modularity/conductance via petgraph on the affected subgraph (shared with P1.7).
- `fn predict_would_refuse(eff: &Effects) -> Option<Refusal>` @ rmc-env::sim::predict — new — for cycle refusals, runs the crate-edge cycle probe (`recursion_check::find_cycles_from`-style DFS over `crate_edges()` of the overlay) to detect a would-be import cycle; folds in `eff.refusal` from the gate pass.

**Key logic / algorithm:**
- **Single source of truth.** The only new architectural move is factoring `CrudEngine::apply` into `compute_effects` (pure) + `persist` (effectful). `apply == persist∘compute_effects`; `simulate == predict∘compute_effects`. Because both verbs funnel through the identical `compute_effects`, the simulator's `would_refuse` and `edit_class`/`affected`/`patch` are byte-identical to what apply would do — it structurally cannot lie (resolves issue #7).
- **Dry-run via uncommitted txn, not a copy.** `predict::from_effects` does **not** `mdb_copy` the working snapshot. It opens one `RwTxn`, applies `eff.patch` (reuse `apply_patch`), runs the affected-set-scoped audits/metrics through the existing read API, then drops the txn so LMDB discards the writes. This keeps `simulate` cheap enough to call as a planning primitive (the design's "cheap lookahead").
- **Deltas reuse existing audits verbatim.** `audit_delta` is computed by calling the same functions the reward path (P1.7) and gate path (P1.6) call — `OpenedSnapshot::{unsafe_audit, mut_static_audit, dead_pub_in_crate}`, `fn_body_audit::fn_body_audit`, `recursion_check::recursion_check` — once on the baseline and once on the overlay, scoped to `AffectedSet` so it stays sub-second. `metric_delta` reuses `OpenedSnapshot::crate_dependency_metric` + the petgraph metrics from P1.7.
- **Cascade = the D2 affected-set, rendered.** `Cascade` is just `eff.affected` (crates + reverse-dep item count from `affected_set()` over reversed `crate_edges`) plus the touched callers from `who_calls`/`usages_by_target`, formatted as `Location`s for the agent's view — the same set `persist` would re-extract.
- **No source write on the simulate path.** `compute_effects` builds `SourceEdit`s in memory and (for refusal gates that need typed HIR, e.g. cycle/visibility checks) uses the warm host's already-loaded `RootDatabase` via a throwaway `set_file_text`+rollback only if a gate truly needs post-edit semantics; default gates run on the `GraphPatch` overlay to avoid touching the host at all.

**Risks / unknowns:**
- **Overlay-txn read API.** `query/*` methods take `&OpenedSnapshot` bound to an `Env`; exposing a read view over an open-but-uncommitted `RwTxn` may require a small lifetime/trait adapter on `OpenedSnapshot` (verify heed allows reads through the same `RwTxn`). If not, fall back to a scratch sub-env populated from the patch.
- **Gate-dry-run coupling.** Some P1.6 hard refusals (cycle introduction, trait coherence) need *post-edit* RA semantics, not just the graph patch — those force a warm-host `set_file_text`+rollback inside `compute_effects`, making `simulate` no longer free of host mutation. Need to confirm which gates are graph-only vs HIR-requiring; HIR-requiring ones make repeated `simulate` calls as costly as a partial apply.
- **Delta baseline drift.** `predict` diffs against the *current* working snapshot; if the agent simulates several ops in a row, each is independent (no stacking). Stacked/what-if-chains are out of scope here.
- **`compute_effects` purity is load-bearing and untested by the type system.** A future contributor adding an IO call inside `compute_effects` silently re-introduces the lie. Mitigate with the differential test (`tests/simulate_matches_apply.rs`) run in CI on every edit class, and a doc-comment contract; consider a `#[must_use]` `Effects` to discourage discarding without `persist`.
- **petgraph metric cost.** modularity/conductance on the affected subgraph is fine; a full-workspace recompute per `simulate` would re-create the latency wall — must scope to `AffectedSet` (shared concern with P1.7).
