**Key logic / algorithm:**
- **Reuse the embedding cache, do not re-embed.** `build_feature_matrix` calls `rmc_graph::graph::embedding_cache::prepare_embeddings_for` + `embed_prepared_embeddings` (or `ensure_embeddings_for`) which already hit `embeddings_by_target` by `content_hash` + `embedder_version`; pack the returned `ResolvedEmbedding.vector`s row-major into a Candle `Tensor` (`Device::cuda_if_available`, falling back to CPU). Dim from `backend.dim()` (1024 default).
- **Item selection is shared with `semantic_overlaps`.** Reuse `enumerate_similarity_seeds` (skips `NodeKind != Item`, `file/span == None`, `::tests::`) so the `Analyze` view sees the same population the existing similarity tool does — no second definition of "what is an item."
- **`Substrate::Embedding`** runs GMM (soft membership → overlapping concepts, e.g. "parse AND cache"); **`Substrate::Structure`** runs spectral on the call/usage `structural_adjacency` (Laplacian fuses graph topology); **`Substrate::Both`** concatenates L2-normalized embedding rows with normalized spectral coordinates before GMM. GMM/spectral both bottom out in `kmeans_pp`.
- **Seeded determinism (right-sized rigor).** One `AnalyzeSeed` per run; `rng_for("kmeans-init")`, `rng_for("gmm-em")` derive independent `ChaCha8Rng` streams so cluster *labels* are reproducible run-to-run. Per plan §"Right-Sized Rigor": no deterministic GPU kernels chased — only the seed is fixed and logged. Cosine geometry stays consistent with `graph::math::cosine` for centroid scoring.
- **is/ought discipline.** Output is pure `Cluster` membership + `crate_spread` (a *fact*: "this concept spans N crates"); the engine attaches no refactor suggestion. `ClusterId` is run-local and not persisted/content-addressed — clusters are perception, regenerated once per episode and cached by the caller (P1.3 step owner), not stored in LMDB.
- **Differential safety net.** Keep `union_find_clusters` (the extracted `build_clusters`) as a cheap CPU fallback and a test oracle: on a tiny fixture the GMM/spectral connected-component structure must align with the threshold union-find result.

**Risks / unknowns:**
- **Candle eigensolver.** `candle-core` 0.10.2 has no first-class symmetric-eig; spectral may need a power-iteration / Lanczos top-k routine hand-rolled on tensors (adds LOC + numerical care) or a small `nalgebra` dependency for the dense eig (verify which is cheaper — `nalgebra` not currently a workspace dep). Marked (verify).
- **`candle-core` is behind the `embeddings-cuda` feature in `rmc-engine`** but is a plain `[workspace.dependencies]` entry, so `rmc-analyze` can depend on it directly; need to confirm CPU-only build works without CUDA for tests (`Device::Cpu` path) (verify).
- **Dense adjacency at workspace scale.** `n×n` `Tensor` is O(n²) memory; fine at ~thousands of items per the plan's scale note, but a large monorepo could blow up — may need sparse spectral later.
- **Cluster count `k`.** GMM/spectral need `k`; auto-selection (BIC / eigengap) is non-trivial — P1.3a ships explicit `k` (or a coarse eigengap heuristic) and defers principled model selection.
- **GMM on 1024-dim with few items** is ill-conditioned (singular covariances); mitigate with diagonal-covariance + variance floor (`DegenerateMatrix` guard), note as a quality risk for the "perception" mandate (bad clusters = astigmatism, plan issue #9).
- **Cross-crate `node_to_crate` cost.** `structural_adjacency` re-walks usages; reuse the `crate_edges`-style single-scan pattern rather than per-pair queries.
