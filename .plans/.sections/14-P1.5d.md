### P1.5d — CRUD — extract_function / extract_trait / inline
**Goal:** Add the three composing-refactor primitives — lift a body sub-span into a new fn + call (extract_function), synthesize a trait from the called-method surface of a set of items (extract_trait), and the reverse of extraction (inline) — on top of the P1.5a apply==rebuild engine.

**Key logic / algorithm:**
- **extract_function span lift.** Resolve `host_fn`'s body with the new `skeleton::source::fn_body_range` (reuses existing `find_item_syntax` + `ast::Fn::body()` from `collect_body_replacements`). Require the requested `span` to fall strictly inside that body (`SpanNotInBody`/`SpanCrossesItem` otherwise). Slice the source bytes for the lifted statements; replace them in `host_fn`'s body with `let (..) = extracted(args);` (or bare `extracted(args);`). The new fn's body text is the lifted slice.
- **Free-variable → params.** Call the new `references_in_span(host_fn, span)` (a span-restricted view over `usages_for_consumer_function`, the same DUP_SORT helper `calls_from` uses); cross-reference against `host_fn`'s `function_signature().params` + locals to build `Capture`s, each emitted as a `Param` with `by_ref`/`mutability`. Values mutated then read after the span force `&mut`; if a capture's type can't be recovered → `CaptureTypeUnknown` (refuse — composition over guessing). Synthesize the new `FunctionSignature` (params = captures, return = single live-out value or `()`); render it with the existing `fallback_function_declaration` shape from skeleton::source.
- **extract_trait surface (THEORY P6).** `surface_from_callsites` does NOT read the items' own declared methods; it builds the trait from *how the items are used*: for each `item` in `items`, `who_calls(item)`/`usages_for_target` → resolve the called member → collect its `function_signature`. The union (deduped by `(name, normalized-sig)`) is the trait surface — exactly "the contract the call sites already depend on." Emit `trait <Name> { <sigs>; }` plus one `impl <Name> for <T>` per source item delegating to the existing inherent method. Coherence (P15) is left to the P1.6 gate, not pre-checked here.
- **inline = extraction reversed.** `plan_inline` reads `target`'s full item via `item_syntax_range` and its `function.body()` slice, enumerates every call site with `who_calls(target)` (each an `EnrichedCallSite` carrying `file`+`start`+`end`), and produces a `Splice` per site replacing the call expression with the substituted body (params bound to call args). Then a `delete(target)` of the now-dead fn. Refuse `Recursive` (target ∈ its own `call_graph`) and `MultiBodyOverload`.
- **Edit application + re-sync.** All three return an `EditPlan::MultiEdit` consumed by P1.5a's `compute_effects`/apply; multi-file call-site rewrites reuse the `rmc-server` `semantic::rename::{RenameEdit, SourceChange}` mechanics (RA `set_file_text` per dirty file). Edit class is **ItemAddRemove** (new/removed `pub`/private fn) under D2 → re-extract via `graph::extract::emit_crate_scoped` for the editing crate + reverse-dep usages/bindings; inline of a cross-crate `pub` fn also re-resolves importers. Each op is wrapped in a D4 `Checkpoint`.

**Risks / unknowns:**
- **Type inference for captures is the hard part** (verify): the graph has `Usage` references and `FunctionSignature` params, but *local* variable types inside a body are not modeled — recovering a capture's type may require an RA `Semantics` type-of query on the warm host, not just LMDB. If unavailable, extract_function degrades to refuse-on-unknown-type, sharply narrowing what it can lift.
- **Borrow/move legality** of a lifted span (a value moved inside the span then used after) is not visible to the syntactic extractor; `BorrowCannotLift` is best-effort and the real check is the P1.6/P1.7 compile gate (RA misses full borrowck per m0-spikes Spike 2 — borrow errors surface only at episode-end cargo).
- **extract_trait "surface from call sites"** assumes the called members resolve to inherent fns with retrievable `function_signature`s; generic/where-bound methods may render incompletely (the `GenericBound::bounds` RA caveat in model.rs — where-clause bounds are a partial view).
- **inline arg-substitution correctness**: textual binding of params to argument expressions can capture-shadow or re-evaluate side-effecting args; a safe lowering may need `let` bindings per arg rather than inlining the expression — bumps LOC if hardened.
- **MultiEdit transactionality**: a new-item + body-splice + N call-site rewrites spanning files must all land or none; relies on the per-op D4 undo-log being correct across files.
