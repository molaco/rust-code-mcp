**Key logic / algorithm:**
- `ContextView` is a pure composition layer over `OpenedSnapshot` read methods — no new graph traversal: `lookup_by_qualified_name` to resolve a `Location` to a `NodeId`; `who_calls`/`calls_from` (graph::query::calls) for caller/callee rows; `module_tree`/`module_dependencies` (graph::query::modules, imports) for Country/Continent scale; `find_root_module_of` + `Node.parent_id` chain for the module path breadcrumb.
- `show_body` toggles the existing skeleton body-strip: hidden = `render_source_item_text`'s `{ /* ... */ }` replacement (collect_body_replacements); shown = raw `slice_range(source, syntax.text_range())`. Body line count for the "Body: hidden (N lines)" summary is `(span.1 - span.0)` newline count from the sliced text. `render_item_source` is the new single-NodeId entry that both paths share, factored out of `SourceCache::render_item`.
- Scale transitions read structure already in the graph: zoom_out walks `Node.parent_id` (item->module->crate); zoom_in descends `children_by_parent` (via `module_tree` depth-1) picking the first/named child. `City`/cluster scale is stubbed to refuse-with-reason until rmc-analyze (P1.3) lands `Cluster`/`ClusterId`; the v3 plan explicitly accepts this hole.
- Runs against the SLOW full-rebuild snapshot (`OpenedSnapshot`, read-only LMDB) per the M1 ordering — Navigate is independent of the P0.2 warm host. No mutation, so no `WorkspaceHost`/checkpoint coupling here.
- `follow_trail` is a thin adapter that turns a hyperedge query result (`ReExportChain.links`, `EnrichedCallSite`) into `Vec<Neighbor>` so the corridor-walk metaphor reuses existing query output verbatim.

**Depends on:** [P0.1] (stable snapshot to read); soft/forward deps on [P1.3] (Cluster/ClusterId for City scale + `salience`/`affinity` fields) and [P1.2] (Description for the `description` field) — both rendered as `Option`/stub so P1.1 ships before them.

**Risks / unknowns:**
- `MapScale::City` and `Anchor::Cluster` reference `ClusterId` from rmc-analyze (P1.3, verify name) — must compile P1.1 without it, so cluster fields are `Option`/feature-gated or a local placeholder type aliased to the real one later. Small re-plumb risk when P1.3 lands.
- `render_item_source` lifting out of `SourceCache::render_item` must not regress the batch `render_crate_skeletons` path (shared `find_item_syntax`/`slice_range`); covered by keeping existing skeleton tests green + a new per-NodeId test.
- "Body line count" + violation summary in the hidden view needs the audit layer (`fn_body_audit`) to attribute findings to a NodeId; until P1.6 wires per-item audits, `audit_status`/violation count is `None`/"unknown" (verify the audit query keys by NodeId).
- `available_ops` list is presentation only here; the authoritative refusal set comes from P1.6 gates — risk of drift if hardcoded, so derive it from scale + node kind, not a static table.
- `OpenedSnapshot` lives in rmc-graph; rmc-env taking it as a borrow is fine for reads, but the eventual working-snapshot (D1) is a different handle — the `ContextView` builder must accept a trait/`&OpenedSnapshot` abstraction so it works against both (verify the working-snapshot also exposes `OpenedSnapshot`-shaped reads).
